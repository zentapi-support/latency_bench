# CLI 延迟测试

用于测试 Codex CLI 的等待时间：从运行命令开始，到完整回答返回为止，用户需要等多久。

## 使用方法

测试 Codex：

```bash
python3 codex_exec_bench.py
```

运行前请确保 Codex CLI 已安装、完成登录并可在终端中直接使用：

```bash
codex --version
```

## 测试配置

- Prompt：`不要调用工具。只输出 20 行连续编号，不要解释。`
- 预热次数：2
- 正式测试次数：10
- 单次超时：180 秒
- 工具调用：禁用

以上配置均硬编码在 `codex_exec_bench.py` 顶部，可直接修改。

## 测试结果

结果保存在：

```text
results-short/
└── codex/
    ├── runs.csv
    ├── summary.json
    └── raw/
```

- `runs.csv`：每次运行的延迟、Token 和状态
- `summary.json`：等待时间中位数、等待时间 P90 和成功率
- `raw/`：CLI 原始 JSONL 输出和错误日志

> 原始输出和错误日志可能包含请求内容或本机信息，公开测试结果前请先检查。

## 指标说明

- **等待时间中位数**：表示通常需要等待多久。一半测试比它快，一半测试比它慢。
- **等待时间 P90**：90% 的测试会在这个时间内返回，用来观察较慢情况下的等待时间。
- **成功率**：能够正常返回完整回答的测试比例。

例如：

```text
等待时间中位数：3.20 秒
等待时间 P90：4.80 秒
成功率：100%
```

这里的等待时间包含 CLI 启动、连接服务和生成回答所需的全部时间。它衡量的是普通用户从运行命令到拿到完整结果的实际等待体验，不是首 Token 延迟（TTFT）。

预热测试不参与统计；中位数、P90 和成功率只使用 10 次正式测试计算。失败或超时的测试计入成功率，但不参与等待时间的分位数计算。
